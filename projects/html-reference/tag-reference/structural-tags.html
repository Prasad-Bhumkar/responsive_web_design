HTML Structural and Semantic Tags Guide

Document Structure

Basic HTML5 Document Structure


<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <meta name="description" content="Page description">
        <meta name="keywords" content="HTML, CSS, JavaScript">
        <meta name="author" content="Author Name">
        <title>Document Title</title>
        <link rel="stylesheet" href="styles.css">
        <script src="script.js" defer></script>
    </head>
    <body>
        <!-- Content goes here -->
    </body>
</html>

Important Notes:

  • The DOCTYPE declaration is mandatory and must be the first line
  • The lang attribute helps with accessibility and SEO
  • Meta tags provide important information for browsers and search engines
  • The viewport meta tag is crucial for responsive design

Semantic Layout Elements

HTML5 Semantic Elements for Page Structure


<header>
    <nav>Navigation menu</nav>
</header>

<main>
    <article>
        <header>Article header</header>
        <section>Article section</section>
        <footer>Article footer</footer>
    </article>

    <aside>Sidebar content</aside>
</main>

<footer>Page footer</footer>

Benefits of Semantic Elements:

  • Improved page structure and accessibility
  • Better search engine optimization (SEO)
  • Easier maintenance and updates

Grouping Elements

Elements for Grouping Content


<div>Generic container element</div>
<span>Inline grouping element</span>

Common Attributes:

  • class: for applying CSS styles
  • id: for unique identification
  • style: for inline CSS styles

Best Practices:

  • Use div and span elements sparingly
  • Prefer semantic elements for better structure
  • Avoid using inline styles; use CSS instead

Lists

Ordered and Unordered Lists


<ul>
    <li>List item 1</li>
    <li>List item 2</li>
</ul>

<ol>
    <li>List item 1</li>
    <li>List item 2</li>
</ol>

Common Attributes:

  • type: for specifying list type (e.g., disc, circle, square)
  • start: for specifying the starting number in an ordered list

Best Practices:

  • Use lists for presenting collections of data
  • Avoid using lists for layout purposes
  • Use CSS to customize list styles
