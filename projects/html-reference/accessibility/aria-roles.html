ARIA Roles Reference

An overview of ARIA roles to enhance web accessibility.

What are ARIA Roles?

ARIA roles define the type of user interface element that an HTML element represents, enhancing accessibility for users with disabilities. They help assistive technologies understand the purpose of elements in a web application.

Common ARIA Roles

  • button - Represents a clickable button.
  • navigation - Represents a navigation section.
  • main - Represents the main content of the document.
  • complementary - Represents content that complements the main content.
  • dialog - Represents a dialog box or modal window.
  • alert - Represents an alert message.

Usage Examples

Here are some examples of how to use ARIA roles in HTML:

                
                    <button role="button">Click Me</button>
                    <nav role="navigation">
                        <a href="#home">Home</a>
                        <a href="#about">About</a>
                    </nav>
                    <div role="dialog" aria-labelledby="dialogTitle">
                        <h2 id="dialogTitle">Dialog Title</h2>
                        <p>This is a dialog box.</p>
                        <button role="button">Close</button>
                    </div>
